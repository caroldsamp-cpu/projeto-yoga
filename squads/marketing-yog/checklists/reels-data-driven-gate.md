# Checklist: Reels Data-Driven Gate (NamaSer + Sadhana)

Use este checklist antes de publicar qualquer Reel.

## 1) Pré-produção (estratégia)

- [ ] Tema do Reel está em um dos eixos com evidência de demanda:
- [ ] Dor/estresse
- [ ] Respiração/presença
- [ ] Corpo/postura
- [ ] A tese semanal está clara em 1 frase.
- [ ] O valor prático do Reel está claro em 1 frase.
- [ ] O Reel reforça a diferenciação NamaSer e a promessa de construir Sadhana (não é genérico).
- [ ] O insight estratégico foi traduzido para voz da Gaby.

## 2) Roteiro (estrutura)

- [ ] Hook tem no máximo 12 palavras.
- [ ] Hook abre com dor real ou contraste forte.
- [ ] Reframe retira culpa e dá direção.
- [ ] Há 1 prática objetiva e aplicável hoje.
- [ ] O fechamento tem convite claro (não pressão).
- [ ] O CTA usa palavra-chave quando fizer sentido.

## 3) Linguagem (voz)

- [ ] Usa termos do DNA da Gaby (`respira`, `observa`, `sente`, `presença`).
- [ ] Evita termos proibidos (`resultado garantido`, `transformação instantânea`, etc.).
- [ ] Texto está acolhedor e sem julgamento.
- [ ] Não usa urgência artificial.
- [ ] Não usa tom confrontativo.

## 4) Conversão (distribuição)

- [ ] CTA principal está definido:
- [ ] comentário com palavra-chave
- [ ] link na bio
- [ ] direct/manual
- [ ] Existe entrega real para quem comentar/chamar no direct.
- [ ] A ação pós-comentário está operacionalmente viável (manual).

## 5) Qualidade final

- [ ] Reel passou no `voice-gate-gaby`.
- [ ] Não há conflito com o anti-manifesto.
- [ ] O conteúdo está alinhado à tese da semana.
- [ ] Publicação foi registrada para análise em `metrics-weekly`.

## 6) Pós-publicação (aprendizado)

- [ ] Registrar métricas básicas (views, likes, comentários, saves, shares).
- [ ] Registrar taxa de palavra-chave nos comentários.
- [ ] Registrar quantidade de DMs iniciadas.
- [ ] Classificar Reel como:
- [ ] top performer
- [ ] médio
- [ ] baixo
- [ ] Atualizar `data/content-map.yaml` com aprendizado da peça.
