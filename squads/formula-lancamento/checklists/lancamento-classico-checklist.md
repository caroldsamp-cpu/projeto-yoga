# Checklist: Lançamento Clássico

**Author:** Érico Rocha + Hugo Rocha (Ignição Digital)
**Module:** Lançamento Clássico
**Pattern:** Zero Invention — 100% Source-Based
**References:** 65 aulas transcritas do módulo Lançamento Clássico

## PRÉ-REQUISITOS

- [ ] Lançamento Semente realizado com pelo menos 1 venda comprovada
- [ ] Oferta definida (pode ser a mesma do Semente, revisada)
- [ ] Gateway de Pagamento escolhido e aprovado (mínimo 2 semanas antes do carrinho)
- [ ] Domínio registrado e ativo
- [ ] Questionário Raio X preenchido completamente (1–3h de dedicação)

[SOURCE: Planejamento - Pré-Requisitos.txt, Planejamento - Gateway de Pagamento.txt]

### Calendário Geral
- [ ] Data de início do Aquecimento de Lançamento definida
- [ ] Data de início e fim da CLI definida
- [ ] Data do CPL1 definida
- [ ] Data do CPL2 definida
- [ ] Data do CPL3 definida
- [ ] Data do Efeito Tsunami definida (1–3 dias antes da abertura)
- [ ] Data e horário de abertura do carrinho definidos
- [ ] Data e horário de fechamento do carrinho definidos
- [ ] Todas as datas no papel / calendário compartilhado

## FASE 1 — AQUECIMENTO DE LANÇAMENTO (14–22 dias)

### Planejamento
- [ ] Formato dos conteúdos definido (vídeo, texto, live, reel)
- [ ] Frequência de publicação definida
- [ ] Lista de temas criada (obstáculos universais + específicos do nicho + atalhos)
- [ ] Títulos revisados: cada título parece que leva a Roma?
- [ ] Calendário editorial completo: data + formato + tema + horário

### Preparação
- [ ] Conteúdos produzidos com antecedência
- [ ] Postagens agendadas onde possível

### Execução
- [ ] Conteúdos publicados conforme calendário
- [ ] Stories e reels postados manualmente nos dias definidos
- [ ] Engagement monitorado (objeções reveladas nos comentários registradas na árvore)

## FASE 2 — CONSTRUÇÃO DE LISTA DE INSCRITOS

### CLI Orgânica — Preparação
- [ ] Página de inscrição criada e publicada
- [ ] Página de obrigado criada e publicada (com vídeo se necessário)
- [ ] Formulários e tags configurados no ESP
- [ ] E-mails de captação escritos com antecedência
- [ ] E-mails agendados no calendário definido (iniciar ~1 semana antes do pré-lançamento)
- [ ] Postagens para redes sociais produzidas e agendadas
- [ ] CTAs inseridos nos ativos: bio do Instagram, foto de capa do Facebook, descrição do YouTube, etc.

### CLI Paga — Planejamento
- [ ] Data de início e término definidas
- [ ] Plataformas de veiculação definidas (Facebook / Google)
- [ ] Meta de total de leads definida
- [ ] Meta de custo por lead (CPL) definida
- [ ] Meta de leads por dia definida
- [ ] Orçamento total definido e validado contra as metas

### CLI Paga — Preparação
- [ ] Roteiros dos criativos escritos
- [ ] Criativos gravados e editados
- [ ] Pixel do Facebook instalado na página de inscrição
- [ ] Global Site Tag do Google instalada na página
- [ ] Conversões configuradas (Facebook + Google)
- [ ] Públicos criados (custom audiences, lookalikes, interesses)
- [ ] Campanhas criadas e programadas para a data de início
- [ ] Fluxo completo testado (anúncio → página → formulário → confirmação)
- [ ] E-mail próprio incluído na lista (para monitorar entrega)

### CLI — Pesquisa com o Avatar
- [ ] Formulário de pesquisa criado (Google Forms recomendado)
- [ ] Formulário incluído no e-mail de confirmação de inscrição
- [ ] Perguntas abertas incluídas (objetivos + obstáculos — sem indução de resposta)
- [ ] Respostas monitoradas ao longo da CLI para enriquecer os scripts

## FASE 3 — PRÉ-LANÇAMENTO

### Arquétipos — Decisão
- [ ] Arquétipo do CPL1 e CPL2 escolhido:
  - [ ] Oportunidade / Oportunidade Amplificada
  - [ ] Jeito Errado / Jeito Certo
- [ ] Arquétipo do CPL3 escolhido:
  - [ ] Perguntas e Respostas
  - [ ] Amostra / Aula Exemplo
  - [ ] Índice / Mapa

[SOURCE: Arquétipos - Introdução.txt]

### Script CPL1
- [ ] Roma scriptada: nome do expert + Roma da série + tema do CPL1
- [ ] Seção de Prova scriptada: metralhadora de provas + empatia antes
- [ ] História scriptada: procedência do método e do expert
- [ ] Conteúdo scriptado: arquétipo escolhido (oportunidade OU jeito errado + por que é errado)
- [ ] Gatilhos mentais inseridos em pontos estratégicos
- [ ] Objeções selecionadas e inseridas onde faz sentido narrativo
- [ ] Antecipação do CPL2 scriptada (criar curiosidade genuína)
- [ ] Fechamento scriptado: call to community + CTA
- [ ] Revisão final: o script vende no CPL1? Se sim, remover a venda.

### Script CPL2
- [ ] Roma do CPL2 scriptada: apresentação + recap da série
- [ ] Novas provas selecionadas (diferentes das do CPL1) + empatia antes
- [ ] Revisão do CPL1 scriptada: resumo atrativo que incentiva (re)assistir
- [ ] Conteúdo scriptado: segunda parte do arquétipo
- [ ] Semente da oferta scriptada: mencionada de forma leve + desapego obrigatório
- [ ] Antecipação do CPL3 scriptada (criar expectativa forte)
- [ ] Fechamento scriptado: revisão do CPL2 + call to community
- [ ] Intensificação de gatilhos verificada: reciprocidade, autoridade, prova social, comunidade
- [ ] Revisão final: parece que está vendendo? Se sim, suavizar.

### Árvore de Objeções — Atualização pré-CPL3
- [ ] Árvore revisada com novas objeções da comunidade e comentários
- [ ] Todas as objeções que CPL1 e CPL2 NÃO mataram identificadas
- [ ] Objeções restantes listadas em ordem de prioridade para o CPL3

### Script CPL3
- [ ] Roma do CPL3 scriptada: recap da série completa (CPL1 + CPL2 + CPL3)
- [ ] Prova + Prova Social scriptadas: depoimentos novos + participação da comunidade
- [ ] Conteúdo scriptado: arquétipo do CPL3 (independente dos anteriores)
- [ ] Oferta completa scriptada:
  - [ ] Entregáveis listados
  - [ ] Benefícios de cada entregável (não basta listar — tem que parecer valioso)
  - [ ] Preço apresentado
  - [ ] Garantia apresentada
  - [ ] Escassez apresentada (tempo, vagas, bônus — tudo real e verificável)
- [ ] Instrução de matrícula scriptada: o que fazer, como, quando, onde
- [ ] Objeções restantes resolvidas dentro do conteúdo e da oferta
- [ ] Fechamento scriptado: revisão de toda a jornada + antecipação do carrinho
- [ ] Revisão final: o avatar sai do CPL3 sem nenhuma objeção?

[SOURCE: Preparação - CPL 1 - Estrutura Geral.txt, Preparação - CPL 2 - Estrutura.txt, Preparação - CPL 3 - Estrutura.txt]

### Infraestrutura do Pré-Lançamento
- [ ] Blog de Lançamento configurado (ambiente fechado com login por e-mail)
- [ ] Estrutura do blog: CPL1 → CPL2 → CPL3 → Vídeo de Vendas (visível na hora certa)
- [ ] Acesso testado end-to-end
- [ ] Comunidade de Lançamento criada na plataforma escolhida
- [ ] Regras da comunidade definidas e publicadas
- [ ] Boas-vindas configuradas

### Gravação dos CPLs
- [ ] CPL1: script lido no espelho 5–7 vezes antes de gravar
- [ ] CPL1: gravado, editado e aprovado
- [ ] CPL2: script lido no espelho 5–7 vezes antes de gravar
- [ ] CPL2: gravado, editado e aprovado
- [ ] CPL3: script lido no espelho 5–7 vezes antes de gravar
- [ ] CPL3: gravado, editado e aprovado
- [ ] CPLs agendados no blog de lançamento nas datas corretas

### Efeito Tsunami — Preparação (fazer ANTES de publicar o CPL1)
- [ ] Script do tsunami escrito (boa notícia + má notícia + escassez + instruções)
- [ ] Página do tsunami criada e publicada (oculta até o momento certo)
- [ ] E-mails de tsunami escritos e agendados
- [ ] Todas as escassez prometidas no tsunami são reais e verificáveis

### Notificações — Preparação
- [ ] Notificações do CPL1 escritas (lembrete antes + lembrete no dia)
- [ ] Notificações do CPL2 escritas
- [ ] Notificações do CPL3 escritas
- [ ] Notificações do Efeito Tsunami escritas
- [ ] Notificações de abertura do carrinho escritas
- [ ] Notificações de fechamento escritas (urgência crescente)
- [ ] Remarketing de pré-lançamento configurado (para inscritos que não abriram os e-mails)

[SOURCE: Preparação - Notificações.txt, Preparação - Efeito Tsunami - Considerações gerais.txt]

### Execução do Pré-Lançamento

**Dia do CPL1:**
- [ ] Publicação do CPL1 verificada (bug check — o sistema agendou corretamente?)
- [ ] E-mail de notificação disparado
- [ ] Stories/Reels postados manualmente
- [ ] Comunidade monitorada ativamente

**Dia do CPL2:**
- [ ] Publicação do CPL2 verificada
- [ ] E-mail de notificação disparado
- [ ] Stories/Reels postados manualmente
- [ ] Novas objeções da comunidade registradas na árvore

**Dia do CPL3:**
- [ ] Publicação do CPL3 verificada
- [ ] E-mail de notificação disparado
- [ ] Stories/Reels postados manualmente
- [ ] Objeções emergenciais da comunidade: incorporar no CPL3 se ao vivo; regravar seção se gravado

**Dias do Efeito Tsunami (1–3 dias antes do carrinho):**
- [ ] Página do tsunami ativada
- [ ] E-mails do tsunami disparados em sequência
- [ ] Postagens nas redes sociais apontando para a página do tsunami
- [ ] Monitoramento de engajamento e dúvidas

## FASE 4 — LANÇAMENTO (Carrinho Aberto)

### Script e Gravação do Vídeo de Vendas
- [ ] Script da Promessa: intenção declarada + definição do produto para quem nunca viu nada
- [ ] Script da Prova: evidências de Roma possível (podem ser repetidas dos CPLs)
- [ ] Script da História: procedência do método (pode ser literalmente a mesma do CPL1)
- [ ] Script da Oferta completa: para quem é + entregáveis + benefícios + como + quando + objeções integradas
- [ ] Script do Preço: custo de oportunidade → preço → comparação → fatiamento → bônus → garantia
- [ ] Script do CTA: chamada direta + comprometimento sem desespero
- [ ] Script lido no espelho antes de gravar
- [ ] Vídeo de vendas gravado e editado

### Página de Vendas
- [ ] Página de vendas configurada no blog de lançamento
- [ ] Vídeo de vendas hospedado na página
- [ ] Botão de compra funcionando
- [ ] Conexão com gateway de pagamento testada
- [ ] Fluxo completo testado: vídeo → botão → checkout → confirmação

### Abertura do Carrinho
- [ ] Página de vendas ativada no horário exato prometido
- [ ] E-mail de abertura disparado para toda a lista
- [ ] Postagem nas redes sociais orgânicas
- [ ] Remarketing pago ativado (para inscritos que ainda não compraram)
- [ ] Comunidade notificada da abertura

### Durante o Carrinho
- [ ] Volume de vendas monitorado em tempo real
- [ ] Dúvidas respondidas na comunidade e por e-mail
- [ ] Padrão do gráfico identificado: J invertido / U / W / J
- [ ] Se Efeito W planejado:
  - [ ] Gatilho do efeito W escolhido (bônus novo / mudança de garantia / parcelamento no boleto)
  - [ ] Anúncio do Efeito W disparado para lista e comunidade
  - [ ] Urgência renovada comunicada

### Fechamento do Carrinho
- [ ] Notificações de fechamento iminente disparadas (últimas 24h, últimas 3h, última 1h)
- [ ] Carrinho fechado no horário exato prometido
- [ ] Página de vendas desativada ou redirecionada para "inscrições encerradas"
- [ ] Confirmação de que nenhuma compra é possível após o fechamento

## CHECKLIST DE QUALIDADE — ERROS COMUNS A EVITAR

### CPL1 e CPL2
- [ ] O "jeito errado" NÃO é "não comprar meu produto"
- [ ] Não existe venda explícita no CPL1
- [ ] No CPL2, a oferta é apenas semeada (levíssima), não apresentada em detalhe
- [ ] O CPL2 "parece" que não está vendendo — mesmo que esteja plantando a semente
- [ ] Cada CPL funciona como episódio independente (alguém que cai de paraquedas entende o contexto)

### CPL3
- [ ] A árvore de objeções foi atualizada ANTES de scriptar o CPL3
- [ ] A escassez prometida é 100% real (nunca artificial)
- [ ] As instruções de matrícula são claras, diretas e sem ambiguidade

### Vídeo de Vendas
- [ ] A promessa não é Roma — é a intenção de explicar tudo para a decisão de compra
- [ ] O produto é explicado do zero (não assume que a pessoa viu os CPLs)
- [ ] O CTA tem comprometimento sem desespero

### Efeito Tsunami
- [ ] O script foi preparado ANTES do CPL1
- [ ] Toda escassez comunicada é real e verificável
- [ ] O tsunami não está tentando compensar uma oferta fraca (oferta fraca = problema anterior, não do tsunami)
