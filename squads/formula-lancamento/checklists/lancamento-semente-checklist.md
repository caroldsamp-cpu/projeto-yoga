## PRÉ-LANÇAMENTO

### Step 1: Pesquisa com o Avatar

**Precondições:**
- [ ] Base e Fundamentos completos (Roma, Avatar, Oferta definidos)

**Execução:**
- [ ] Contatei pessoas do meu nicho para pesquisa
- [ ] Coletei dores e obstáculos na voz do avatar (não na minha voz)
- [ ] Coletei vocabulário real que o avatar usa para descrever seus problemas
- [ ] Identifiquei as 3-5 principais objeções do avatar

**Output Validado:**
- [ ] `pesquisa-avatar.txt` criado com respostas reais coletadas

---

### Step 2: Decisão sobre o Produto

**Execução:**
- [ ] Decidi: vendo antes de gravar OU tenho produto já pronto

**Se vender antes de gravar:**
- [ ] Produto completamente esquematizado (todos os módulos e aulas planejados)
- [ ] Datas de entrega de cada módulo já definidas para anunciar no webinário

**Se produto já está gravado:**
- [ ] Produto funcional e completo para entrega imediata

**Output Validado:**
- [ ] Decisão documentada e esquema do produto pronto

---

### Step 3: Script de Vendas Construído

**Precondições:**
- [ ] Pesquisa com avatar feita (Step 1)
- [ ] Oferta definida (Base e Fundamentos)

#### 3.1 Roma (DSA)
- [ ] Roma está como **Destino** (não veículo/método)
- [ ] Roma é **Simples** — cabe em 1 frase curta
- [ ] Roma é **Atrativa** para o avatar (não só para mim)
- [ ] Roma abre o script nos primeiros 2 minutos

#### 3.2 Prova
- [ ] Tenho pelo menos 1 caso de resultado real para apresentar
- [ ] Prova é específica (nome, situação, resultado)

#### 3.3 História
- [ ] História do expert escrita — jornada de onde veio até hoje
- [ ] História conecta emocionalmente o avatar ao expert/método

#### 3.4 Conteúdo / Ensino
- [ ] Conteúdo conectado à Roma ANTES de ensinar o "como"
- [ ] Conteúdo entrega valor real (não é só promessa)
- [ ] Uso de histórias e/ou analogias no ensino

#### 3.5 Oferta
- [ ] Entregáveis apresentados claramente
- [ ] Preço apresentado com comparável (custo de NÃO ter a Roma)
- [ ] Garantia apresentada
- [ ] Bônus apresentados (se houver)

#### 3.6 Quebra de Objeções
- [ ] Top 3-5 objeções da pesquisa com avatar respondidas no script

#### 3.7 CTA
- [ ] Instrução clara de como comprar (link, botão, etc.)
- [ ] Prazo de fechamento do carrinho anunciado (terça-feira)
- [ ] CTA repetido ao menos 2x no webinário

**Output Validado:**
- [ ] Script completo escrito e ensaiado ao menos 1 vez

---

### Step 4: Elementos Técnicos Preparados

**Execução:**
- [ ] Página de inscrição do webinário funcionando (com formulário + tag)
- [ ] Plataforma de transmissão ao vivo configurada e testada
- [ ] Página de vendas funcionando
- [ ] Gateway de pagamento configurado e testado (fiz uma compra teste)
- [ ] Sequência de e-mails de aquecimento preparada
- [ ] Sequência de e-mails de carrinho aberto preparada (sex, dom, ter)
- [ ] Anúncios de captação criados e aprovados (se pago)

**Guardrail:**
- [ ] Todos os elementos testados ANTES de D-14 (não na véspera)

## FASE 1: Construção da Lista (D-14 ao Webinário)

### Step 5: Captação Paga

**Precondições:**
- [ ] Anúncios aprovados (Step 4)
- [ ] Data do webinário definida no calendário

**Execução:**
- [ ] Anúncios ativados no dia D-14 (14 dias antes do webinário)
- [ ] Monitoramento diário de custo por lead
- [ ] Ajustes de criativos feitos se CPL acima do esperado

**Output Validado:**
- [ ] Lista de inscritos crescendo de forma consistente

---

### Step 6: Captação Orgânica (apenas se tem audiência)

**Execução:**
- [ ] Chamadas postadas nas plataformas onde tenho audiência
- [ ] Iniciado no mesmo dia da captação paga

**Guardrail:**
- [ ] Expectativa calibrada: orgânico = bônus, não principal

**Decision:**
- [ ] IF zero audiência → Step ignorado (ok)
- [ ] IF tem audiência → usado como complemento ao pago

## FASE 3: Webinário — Execução (Quinta-feira)

### Step 8: Script Executado

**Execução (sequência obrigatória):**
- [ ] Roma (DSA) apresentada nos primeiros 2 minutos
- [ ] Prova apresentada após a Roma
- [ ] História do expert contada
- [ ] Conteúdo com valor real entregue (conectado à Roma)
- [ ] Oferta completa apresentada (entregáveis + preço + garantia + bônus)
- [ ] Principais objeções respondidas
- [ ] CTA claro dado com link de compra
- [ ] Prazo de fechamento (terça) anunciado
- [ ] Carrinho aberto ao final do webinário

**Guardrail:**
- [ ] A oferta FOI feita durante o webinário (não é aula sem oferta)
- [ ] Webinário NÃO foi executado como "aula comum" sem script

### Step 9: Sequência de Fechamento Enviada

**Calendário:**

## PÓS-LANÇAMENTO

### Step 10: Entrega do Produto

**Precondições:**
- [ ] Carrinho fechado
- [ ] Alunos matriculados

**Execução:**
- [ ] Boas-vindas enviadas com cronograma de entrega
- [ ] Cada aluno acompanhado de perto (turma pequena)
- [ ] Validado: o método está funcionando para os alunos?
- [ ] Over delivery feito onde necessário (entreguei além do prometido)
- [ ] Testemunhos coletados dos alunos que tiveram resultado

**Output Validado:**
- [ ] Alunos com progresso documentado
- [ ] Testemunhos coletados para usar no Lançamento Clássico

---

### Step 11: Debriefing Feito

**Precondições:**
- [ ] Carrinho fechado
- [ ] Cabeça fria (alguns dias após o lançamento)

**Execução:**
- [ ] 1 dia inteiro reservado no calendário para o debriefing
- [ ] Documento de debriefing preenchido com máximo detalhe
- [ ] Por cada fase do lançamento:
  - [ ] O que deu certo? → documentado
  - [ ] O que deu errado? → documentado
- [ ] Ações de melhoria para o próximo lançamento listadas

**Guardrail:**
- [ ] Debriefing NÃO feito no meio do lançamento
- [ ] Debriefing NÃO deixado para meses depois
- [ ] Documento preenchido com profundidade (não "nas coxas")

**Output Validado:**
- [ ] `debriefing-ls-1.txt` criado e completo
