# SOP: Base e Fundamentos da Fórmula de Lançamento

**Author:** Érico Rocha + Hugo Rocha (Ignição Digital)
**Module:** Base e Fundamentos
**Pattern:** Zero Invention — 100% Source-Based Extraction
**Purpose:** Passo a passo operacional para configurar os fundamentos antes do primeiro lançamento

---

## Visão Geral do Processo

```
NICHO → AVATAR → ROMA → MÉTODO → OFERTA → CONTEÚDO → CALENDÁRIO → CICLO INICIAL
```

Cada etapa é pré-requisito para a próxima. Não pule etapas.

---

## PARTE 1: Fundamentos Estratégicos

### Step 1: Definir o Nicho

**Cognitive Type:** Judgment
**Executor:** Human (Expert/Lançador)
**Precondition:** Nenhuma — ponto de partida

**Ação:**
1. Identifique o mercado/área onde você pretende atuar
2. Avalie se o nicho é grande demais (ex: "marketing", "saúde")
3. Aplique a estratégia de subnicho: reduza para um segmento específico e defensável

**Heurística de Subnicho:**

> "Como é que você geralmente foge desse tipo de coisa? A resposta é você subnichar."
> — Hugo Rocha, [SOURCE: Nicho.txt]

**Exemplo:**
```
Marketing → Marketing Digital → Lançamentos Digitais → Fórmula de Lançamento
```

**Guardrail:**
- Nicho muito amplo = briga de foice com gigantes
- Nicho muito estreito = audiência insuficiente para 6em7

**Output:** `nicho_definido.txt` — nicho e subnicho documentados

**Decision Rule:**
- IF nicho tem concorrentes tipo "faculdades + agências gigantes" THEN subnichar mais
- IF consegue definir Roma única sem comparação direta com concorrentes THEN nicho está bom

[SOURCE: Nicho.txt, Preço.txt]

---

### Step 2: Definir o Avatar

**Cognitive Type:** Analysis + Empathy
**Executor:** Human
**Precondition:** Nicho definido (Step 1)

**Ação:**
1. Descreva detalhadamente o cliente ideal: quem é, o que faz, o que sente
2. Identifique as dores, medos, desejos e obstáculos do avatar em relação ao nicho
3. Mapeie o vocabulário que o avatar usa para descrever seus problemas
4. Documente em formato de perfil escrito

**Guardrail:**
- O avatar deve ser uma pessoa real e específica, não um "público geral"
- Use o Questionário Raio X para formalizar (Step 7)

**Output:** `avatar_profile.txt` — perfil completo documentado

[SOURCE: Avatar.txt, Nicho Avatar e Roma.txt]

---

### Step 3: Definir a Roma

**Cognitive Type:** Synthesis + Creativity
**Executor:** Human
**Precondition:** Avatar definido (Step 2)

**Ação:**
1. Responda: "Qual é a transformação específica que eu vou prometer ao meu avatar?"
2. Formule em linguagem que o avatar usa (não em linguagem técnica do expert)
3. Valide: a Roma é replicável para outras pessoas seguindo um método?
4. Escreva a Roma em 1-2 frases concisas

**Critérios da Roma:**

> "Tem que ser um passo a passo, tem que ser uma coisa que seja fácil de se entender e tem que ser replicável."
> — Hugo Rocha, [SOURCE: Formalização do Método.txt]

**Teste de Roma:**
- IF "Essa aula ajuda o avatar a chegar em Roma?" → YES = conteúdo válido
- IF "Essa copy leva o avatar mais perto de Roma?" → YES = copy válida
- IF Roma depende de talento único irreplicável → VETO — redefina

**Importância:**

> "Se você acerta na sua Roma, você praticamente tá 60% do caminho andado pra chegar no seu 6 em 7."
> — Hugo Rocha, [SOURCE: Roma.txt]

**Output:** `roma_definida.txt` — transformação prometida em 1-2 frases

[SOURCE: Roma.txt, Roma Transformação.txt]

---

### Step 4: Formalizar o Método

**Cognitive Type:** Synthesis + Memory
**Executor:** Human (Expert)
**Precondition:** Roma definida (Step 3)

**Ação:**
1. Mapeie o passo a passo que leva o avatar da situação atual até a Roma
2. Dê nome ao método (ex: "Fórmula de Lançamento")
3. Documente em formato de documento escrito (não na cabeça)
4. Valide: qualquer pessoa determinada consegue seguir e replicar?

> "Formalização do seu método é um passo a passo replicável para uma pessoa ou o seu avatar chegar até a Roma."
> — Hugo Rocha, [SOURCE: Formalização do Método.txt]

**Guardrail:**
- Método que depende de "dom especial" = não replicável = VETO
- Método sem documentação = risco de inconsistência no ensino

**Output:** `metodo_formalizado.txt` — passo a passo documentado

[SOURCE: Formalização do Método.txt]

## PARTE 2: Oferta

### Step 5: Montar a Oferta

**Cognitive Type:** Synthesis + Judgment
**Executor:** Human
**Precondition:** Método formalizado (Step 4)

A oferta tem 4 componentes obrigatórios:

#### 5.1 Entregáveis (Formato do Veículo)

Defina como você vai entregar o método ao avatar:

| Formato | Eficiência para 6em7 |
|---------|---------------------|
| Curso online | Alta — mais escalável |
| Consultoria privada | Média — requer mais horas |
| Mentoria em grupo | Média-Alta |
| Produto físico | Média |
| Serviço | Baixa — difícil de escalar |

> "Um curso online é muito mais fácil fazer um 6 em 7. Do que fazer com consultoria privada. Não é que não dê pra fazer, mas um geralmente é mais eficiente que o outro."
> — Hugo Rocha, [SOURCE: Entregáveis.txt]

**Output:** Formato definido + conteúdo dos módulos esboçado

#### 5.2 Preço

1. Identifique comparáveis — quanto custa o avatar NÃO ter a Roma?
2. Compare com outros produtos do mercado (só se Roma não for única)
3. Defina formas de pagamento (parcelas, boleto, cartão)

> "Você pode comparar com outras coisas que a pessoa teria que fazer pra conquistar o mesmo objetivo que ela tá conquistando com seu curso."
> — Hugo Rocha, [SOURCE: Preço.txt]

**Heurística de comparável:**
```
Custo de NÃO ter Roma > Preço do produto
```

**Guardrail:**
- IF Roma é única e diferenciada → ENTÃO não precisa se comparar com concorrentes
- IF Roma é genérica → ENTÃO levar em conta preços do mercado na precificação

**Output:** Preço definido + formas de pagamento

#### 5.3 Garantia

Define como o avatar pode se sentir seguro para comprar sem risco. Remove objeção de compra.

**Output:** Política de garantia definida

#### 5.4 Bônus

Complementos que aumentam o valor percebido da oferta principal.

**Output:** Lista de bônus (se houver)

**Pré-requisito para Step 6:**

> "Você vai marcar a data do seu lançamento semente no momento em que você tiver com a sua oferta pronta."
> — Hugo Rocha, [SOURCE: Calendário.txt]

[SOURCE: Entregáveis.txt, Preço.txt, Garantia.txt, Bônus.txt]

## PARTE 3: Audiência

### Step 6: Criar Sistema de Conteúdo Raiz

**Cognitive Type:** Creativity + Accountability
**Executor:** Human (Expert + Lançador)
**Precondition:** Roma definida (Step 3)

#### 6.1 Entender o Papel do Lançador no Conteúdo

O maior erro do expert: ensinar o "como" sem conectar à Roma.

> "Antes de ensinar o como, você precisa conectar o como à Roma. O começo de um conteúdo, a envelopagem do conteúdo tem que estar o como conectado com a Roma."
> — Hugo Rocha, [SOURCE: Conteúdo Raiz - Roteiro.txt]

**Regra:**
```
Antes de ensinar técnica → mostrar COMO técnica conecta com a Roma do avatar
```

**Exemplo real:**
- ❌ Errado: "Exercício que aumenta força nos membros inferiores"
- ✅ Certo: "Como esse exercício vai deixar seu corpo 20 anos mais jovem" [antes de ensinar o exercício]

[SOURCE: Conteúdo Raiz - Roteiro.txt]

#### 6.2 Definir Linhas Editoriais

> "É você agrupar um conteúdo que tem o mesmo formato e que fala sobre um mesmo tema."
> — Hugo Rocha, [SOURCE: Conteúdo Raiz - Linhas editoriais.txt]

**Ação:**
1. Defina 2-3 linhas editoriais para o seu nicho
2. Cada linha tem formato pré-definido + tema central
3. Comece a criar dentro dessas linhas regularmente

**Exemplos de linhas editoriais:**
- Consultoria pública (formato tipo "747" do Érico)
- Podcast com cases/entrevistas
- Bastidores do processo
- Mão na massa (implementação ao vivo)
- Entrevistas

#### 6.3 Regra de Consistência vs Qualidade

> "Publicar dois conteúdos por semana [consistentemente] é muito melhor do que publicar oito conteúdos numa semana e passar três semanas sem publicar."
> — Hugo Rocha, [SOURCE: Conteúdo Raiz - Linhas editoriais.txt]

**Fórmula do bom conteúdo:**
```
Qualidade = quão relacionado à Roma
Consistência = frequência regular de publicação
Sucesso = Qualidade × Consistência
```

#### 6.4 Ferramentas de Ensino

Para aumentar retenção e engajamento no conteúdo:

1. **Histórias** — conecta emocionalmente, aumenta retenção
2. **Analogias** — facilita compreensão de conceitos complexos

> "Você reparar o Érico também, cara, ele é mestre em usar analogia. O tempo inteiro ele busca analogia para ensinar qualquer coisa."
> — Hugo Rocha, [SOURCE: Conteúdo Raiz - Roteiro.txt]

**Output:** Linhas editoriais definidas + primeiro conteúdo criado

[SOURCE: Conteúdo Raiz - Linhas editoriais.txt, Conteúdo Raiz - Roteiro.txt]

### Step 7: Preencher Questionário Raio X

**Cognitive Type:** Memory + Analysis + Accountability
**Executor:** Human
**Precondition:** Steps 1-6 concluídos

**O que documentar:**

| Seção | Conteúdo |
|-------|----------|
| **Negócio** | Nicho, Avatar, Roma, Método, Oferta |
| **Competidores** | Quem são, o que oferecem, diferencial seu |
| **Produtos** | O que você vai vender (atual e futuro) |
| **Audiência** | Presença digital atual — redes, tamanho |
| **Listas** | E-mail list e outras listas existentes |

> "Ao documentar você tem clareza, você sempre pode voltar. Pode ser que você precise de uma segunda opinião de alguém."
> — Hugo Rocha, [SOURCE: Questionário Raio X.txt]

**Guardrail:**

> "Se você jogar, fazer os negócios nas coxas, só pra cumprir, não espere resultados satisfatórios."
> — Hugo Rocha, [SOURCE: Questionário Raio X.txt]

**Output:** `questionario-raio-x.txt` — documento completo e honesto

[SOURCE: Questionário Raio X.txt]

## PARTE 4: Planejamento

### Step 8: Montar Calendário de Lançamentos

**Cognitive Type:** Judgment + Accountability
**Executor:** Human
**Precondition:** Oferta pronta (Step 5)

**Ação:**
1. Marque a data do **Lançamento Semente** (quando a oferta estiver pronta)
2. Após a 1ª venda no Semente, marque as datas dos **7 Lançamentos Clássicos**
3. Espaçamento: 45-60 dias entre cada lançamento clássico

> "Essas oito datas são as datas mais importantes que você precisa colocar no calendário."
> — Hugo Rocha, [SOURCE: Calendário.txt]

**As 8 Datas:**
```
Data 1: Lançamento Semente (ciclo inicial)
Data 2: Lançamento Clássico 1 (~2 meses após Semente)
Data 3: Lançamento Clássico 2 (~2 meses após LC1)
...
Data 8: Lançamento Clássico 7 (~2 meses após LC6) → Potencial 6em7
```

**Decision Rules:**

```
IF oferta não está pronta → NÃO marque data do Semente ainda
IF fez 1ª venda no Semente → IMEDIATAMENTE entre nos ciclos evolutivos
IF espaçamento < 45 dias → risco de resultado abaixo do potencial
IF espaçamento > 60 dias → está atrasando o 6em7
IF data cai em feriado/natal → ajuste para semana anterior ou posterior
```

> "Dois meses. Nesses dois meses, você vai ter tempo de entregar o produto para quem comprou no semente, validar sua entrega e continuar postando conteúdo."
> — Hugo Rocha, [SOURCE: Calendário.txt]

**Output:** `calendario-lancamentos.txt` — 8 datas definidas no calendário

[SOURCE: Calendário.txt, Ciclos Evolutivos.txt]

## Regras de Decisão Globais (Base e Fundamentos)

| Condição | Ação |
|----------|------|
| Roma não está clara | STOP — defina Roma antes de avançar |
| Nicho muito amplo | Subnicar antes de continuar |
| Método não está documentado | Documente antes de criar conteúdo |
| Oferta não está pronta | Não marque data de lançamento |
| 1ª venda no Semente realizada | IMEDIATAMENTE entre nos ciclos evolutivos |
| Se viciar no Semente | ALERTA — está atrasando o 6em7 |
