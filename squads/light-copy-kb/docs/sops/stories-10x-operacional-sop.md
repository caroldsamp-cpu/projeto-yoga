## Escopo

Este SOP cobre os sistemas operacionais que suportam o metodo Stories 10x: arquitetura de conteudo Instagram, vendas via SPIN no inbox, crescimento de seguidores por impulsionamento, trafego perpetuo, automacoes, e debriefing.

**Usa-se quando:** Precisa operar a maquina S10X completa (nao apenas criar uma sequencia isolada)
**NAO usa-se quando:** Criando uma unica sequencia de stories (usar SOP Core)

| Sinal | Priorizacao |
|-------|------------|
| Texto longo e detalhado | ALTA |
| Objecoes especificas sobre produto | MUITO ALTA (ja esta considerando) |
| Pergunta direta sobre produto/preco | PRONTO PARA COMPRAR |

**GATE:** Sem triagem = gastando tempo com leads frios enquanto quentes esfriam.

> Fonte: [SOURCE: S10X-Operacional > Inbox lucrativo]

### Step 2.2: Executar SPIN Completo

**Descricao:** Conduzir conversa de venda no inbox usando o framework SPIN + Hook Question.

| Campo | Valor |
|-------|-------|
| cognitive_type | Venda consultiva |
| automation_potential | Baixo (requer julgamento humano) |
| executor | Humano (vendedor/comunicador) |
| precondition | Lead identificado como quente (Step 2.1) |
| output | Venda fechada OU lead nurturado |
| guardrails | 80% ouvir, 20% falar; NUNCA empurrar produto sem diagnostico |
| decision_rule | RD-04 |

**Fluxo SPIN:**

**S — Situacao (Diagnostico):**
Perguntas para entender o estado atual.
- "Qual e seu [metrica] hoje?"
- "Ha quanto tempo voce [situacao]?"

**P — Problema:**
Cavar a dor.
- "Que [solucao] voce ja tentou? Por que nao funcionou?"
- "O que mais te frustra sobre [situacao]?"

**I — Implicacao:**
Amplificar urgencia.
- "O que acontece se isso continuar por mais 5 anos?"
- "Como isso afeta [area importante da vida]?"

**N — Necessidade:**
Levar a conclusao.
- "Voce acredita que deveria investir mais nisso?"
- "Como voce imagina sua vida depois de resolver isso?"

**Hook Question (Invencao Ladeira):**
- "Sabendo de tudo isso, qual a conclusao mais obvia?"
- Se compra → "Me mande o comprovante para eu garantir que voce foi bem recebido"
- Se nao compra → oferecer bonus, agendar call, trial 7 dias

**Pressuposicao do Sim:** Se a pessoa respondeu, ela QUER comprar. Aja como se a venda ja estivesse acontecendo.

> Fonte: [SOURCE: S10X-Operacional > Inbox lucrativo] + [SOURCE: S10X-Operacional > Zoom tira-duvidas]

### Step 2.3: 3 Nao-Facas

| # | Nao-Faca | Por que |
|---|----------|---------|
| 1 | Nao tenha medo de parecer "malicioso" | E ajuda genuina, nao manipulacao |
| 2 | Nao tenha medo de VENDER | Muitos fazem tudo certo e congelam no fechamento |
| 3 | Nao insista demais | Se claramente disse nao, respeite. Insistencia excessiva queima marca |

| automation_potential | Medio |
| executor | Gestor de trafego |
| precondition | Pagina de vendas pronta; pixel instalado |
| output | 1 campanha CBO com 3 ad sets (hot, warm, cold) |
| guardrails | CBO com minimo de gasto por ad set; 3 criativos por ad set (iniciantes) |
| decision_rule | RD-08 |

**Estrutura CBO:**
- 1 campanha com orcamento no nivel de campanha
- Ad Set 1: HOT (compradores, checkout, engajados)
- Ad Set 2: WARM (visitantes, viewers, email)
- Ad Set 3: COLD (lookalikes, interesses)
- 3 criativos por ad set

## Regras de Decisao

| RD-02 | Ritmo semanal indefinido | Min 3 sequencias S10X/semana + 1 right hook (venda) |
| RD-03 | Inbox lotado, quem priorizar? | Texto longo > objecoes > perguntas sobre produto > emojis |
| RD-04 | Como vender no inbox? | SPIN: Situacao → Problema → Implicacao → Necessidade → Hook Question |

| RD-06 | Qual objetivo do impulsionamento? | SEMPRE Visitas ao Perfil; NUNCA engajamento ou cliques |
| RD-07 | Custo/seguidor acima de R$2? | Conteudo nao e bom → criar novo; trocar audiencia quando fica caro |
| RD-08 | Estrutura de campanha perpetuo | 1 CBO, 3 ad sets (hot/warm/cold), 3 criativos cada |
| RD-09 | Distribuicao de budget perpetuo | Iniciante: 95%+5%; Avancado: 15/10/60/5/10 |
| RD-10 | Qual automacao implementar? | Priorizar por ROI: ManyChat > Email > Dashboard > Cupom > Renovacao |
