## Escopo

Este SOP cobre o processo completo de criacao e publicacao de uma sequencia de Instagram Stories usando o metodo Stories 10x — desde a escolha do tema ate o debriefing pos-publicacao.

**Usa-se quando:** Precisa criar qualquer sequencia de Stories (engajamento, venda, aquecimento, evento, etc.)
**NAO usa-se quando:** O conteudo e para Reels/Feed (usar SOP de copy do M4) ou para email/pagina de vendas

---

## Parte 1: Planejamento da Sequencia

### Step 1.1: Escolher Tema [GATE]

**Descricao:** Selecionar o tema da sequencia usando uma das 4 fontes de ideias.

| Campo | Valor |
|-------|-------|
| cognitive_type | Decisao estrategica |
| automation_potential | Baixo (requer criatividade humana) |
| executor | Humano (comunicador) |
| precondition | Banco de ideias com 70+ temas (ver Ideias Fortes e Infinitas) |
| output | 1 tema definido |
| guardrails | Tema deve gerar conversa, nao ser tutorial generico |
| decision_rule | RD-01 |

**4 Fontes de Temas:**

1. **Categorias** — 5 categorias do nicho × ideias cada
2. **Situacoes de Identificacao** — Momentos do dia-a-dia que a audiencia se identifica
3. **Audiencia/Urgencia Oculta** — Problemas que a audiencia tem mas NAO busca pelo nome
4. **Tema Livre** — Ideias aleatorias sem framework

**GATE:** Sem tema definido = sem sequencia. O tema e o combustivel de tudo.

> Fonte: [SOURCE: S10X-Core > Ideias fortes e infinitas]

### Step 1.2: Definir Tipo de Sequencia

**Descricao:** Escolher entre os 12 tipos de sequencia com base no objetivo.

| Campo | Valor |
|-------|-------|
| cognitive_type | Classificacao |
| automation_potential | Medio (app pode sugerir) |
| executor | Humano com apoio do App S10X |
| precondition | Tema definido (Step 1.1) |
| output | Tipo de sequencia selecionado |
| guardrails | 90% das sequencias devem ser engajamento/consciencia/caixinha |
| decision_rule | RD-02 |

**12 Tipos disponiveis:**
1. Venda Produto Digital (alta cadencia)
2. Venda Produto Fisico (baixa-media cadencia)
3. Consciencia de Produto (alta cadencia)
4. Engajamento Puro (variavel)
5. Publicidade (media)
6. Promocao de Parceiro (media)
7. Aquecimento para Pico de Vendas (SEMPRE alta)
8. Pre-Abertura (alta)
9. Caixinha por Tema (baixa)
10. Caixinha Geral (baixa)
11. Evento (alta)
12. Divulgacao de Conteudo (baixa)

> Fonte: [SOURCE: S10X-Core > A logica das sequencias]

### Step 1.3: Definir Cadencia e Data

**Descricao:** Configurar ritmo de publicacao e agendar.

| Campo | Valor |
|-------|-------|
| cognitive_type | Planejamento |
| automation_potential | Alto (app gerencia) |
| executor | Humano com App S10X |
| precondition | Tipo definido (Step 1.2) |
| output | Cadencia (alta/baixa) + data definida |
| guardrails | Aquecimento e Pre-Abertura SEMPRE alta cadencia |
| decision_rule | RD-03 |

**Cadencia vs. Frequencia:**
- **Frequencia** = quantos DIAS por semana faz stories
- **Cadencia** = ritmo/intensidade DENTRO de um unico dia
- Alta cadencia = postando muitas vezes ao longo do dia
- Baixa cadencia = poucos stories manha + noite

> Fonte: [SOURCE: S10X-Core > A logica das sequencias]

## Parte 2: Construcao da Sequencia

### Step 2.1: Escrever Copy dos Stories [GATE]

**Descricao:** Escrever o texto de cada story da sequencia, comecando pelo esqueleto (primeiro, ultimo, 5 dispositivos) e preenchendo o meio.

| Campo | Valor |
|-------|-------|
| cognitive_type | Criacao |
| automation_potential | Baixo (requer criatividade) |
| executor | Humano (copywriter/comunicador) |
| precondition | Tema + Tipo + Cadencia definidos |
| output | Texto de 15+ stories |
| guardrails | Primeiro story DEVE ter CTA; escrever copy primeiro, identificar dispositivos depois |
| decision_rule | RD-04 |

**Ordem de construcao:**
1. Escreva o **primeiro story** (com CTA obrigatorio: enquete, caixinha, ou inbox)
2. Escreva o **ultimo story** (com CTA final)
3. Escreva os **5 stories-chave** com dispositivos planejados
4. Preencha o **meio** (pode ser improvisado com base no inbox)

**GATE:** Sequencia sem copy = nao publicar. O copy define a qualidade da sequencia.

> Fonte: [SOURCE: S10X-Core > Criando as sequencias]

### Step 2.2: Selecionar e Aplicar Dispositivos [GATE]

**Descricao:** Identificar quais dispositivos de engenharia social estao presentes (ou devem ser adicionados) em cada story.

| Campo | Valor |
|-------|-------|
| cognitive_type | Analise + Aplicacao |
| automation_potential | Medio (app checa minimos) |
| executor | Humano com validacao do App |
| precondition | Copy escrito (Step 2.1) |
| output | 5+ stories com dispositivos identificados |
| guardrails | Minimo 5 dispositivos na sequencia; se 0 encontrados → story fraco; se 3+ por story → "pega fogo" |
| decision_rule | RD-05 |

**Processo:**
1. Releia cada story
2. Identifique quais dos 37 dispositivos estao presentes
3. Se nenhum → reescreva ou adicione dispositivos
4. Marque cada story com os dispositivos usados
5. Verifique que o minimo de 5 foi atingido

**Dispositivos mais frequentes por tipo de sequencia:**

| Tipo Sequencia | Dispositivos Prioritarios |
|---------------|--------------------------|
| Engajamento Puro | Desabafo, Historia com Gancho, Piada Interna, Voce Sabia |
| Venda Digital | Ansiedade pela Abertura, Abertura de Carrinho, Print de Resultado, Levante a Mao |
| Aquecimento | Tarja de Curiosidade, Espetacularizacao, Cultura de Resultados |
| Caixinha por Tema | Caixinha por Tema, BI Apurado, Conversa sem Privacidade |
| Evento | Combustivel Extra, Desafio/Gincana, Espetacularizacao |

**GATE:** Menos de 5 dispositivos = sequencia fraca. Dispositivos sao o motor do engajamento.

> Fonte: [SOURCE: S10X-Core > Criando as sequencias] + [SOURCE: S10X-Core > Dispositivos Partes 1-4]

### Step 2.3: Definir CTAs por Story

**Descricao:** Atribuir chamadas para acao em cada story que requer interacao.

| Campo | Valor |
|-------|-------|
| cognitive_type | Planejamento tatico |
| automation_potential | Alto (app valida minimos) |
| executor | Humano com App S10X |
| precondition | Dispositivos aplicados (Step 2.2) |
| output | 5+ stories com CTAs definidos |
| guardrails | Primeiro story OBRIGATORIAMENTE com CTA; minimo 5 CTAs na sequencia |
| decision_rule | RD-06 |

**3 Tipos de CTA:**
- **Enquete (Poll)** — Sim/Nao; algoritmo conta interacao por story
- **Resposta no Inbox** — Mais poderoso; gera conversa + inteligencia de mercado
- **Caixinha de Perguntas** — Perguntas abertas; tema especifico ou geral

**Regra do Algoritmo:** O algoritmo valoriza interacoes POR STORY, nao por sequencia. Stories sem CTA ate o #5 = stories 1-4 tem zero interacoes = algoritmo ve como ruins.

> Fonte: [SOURCE: S10X-Operacional > Como utilizar dispositivos no App]

### Step 2.4: Inserir Links de Produto (se aplicavel)

**Descricao:** Para sequencias de venda, consciencia, ou aquecimento — inserir links trackeados.

| Campo | Valor |
|-------|-------|
| cognitive_type | Execucao tecnica |
| automation_potential | Alto (links pre-cadastrados no App) |
| executor | App S10X ou Humano |
| precondition | Produtos cadastrados no App com UTMs |
| output | Links inseridos nos stories apropriados |
| guardrails | Usar Link Oculto (D36): "Assista essa aula" > "Compre meu curso" |
| decision_rule | RD-07 |

> Fonte: [SOURCE: S10X-Core > Dispositivos Parte 4] + [SOURCE: S10X-Operacional > Cadastrando produtos]

## Parte 3: Publicacao e Monitoramento

### Step 3.1: Publicar Sequencia

**Descricao:** Publicar os stories na ordem planejada, mantendo cadencia definida.

| Campo | Valor |
|-------|-------|
| cognitive_type | Execucao |
| automation_potential | Medio (pode ser delegado a equipe) |
| executor | Humano (comunicador ou equipe treinada) |
| precondition | Sequencia completa e preenchida no App (status = Preenchido) |
| output | Stories publicados; status = Publicado |
| guardrails | Manter cadencia; primeiro story com CTA; nao mudar ordem |
| decision_rule | — |

**Status no App:** Planejando → Preenchido → **Publicado**

### Step 3.2: Monitorar Inbox em Tempo Real

**Descricao:** Durante a publicacao, monitorar mensagens que chegam no inbox e adaptar stories do meio da sequencia.

| Campo | Valor |
|-------|-------|
| cognitive_type | Improvisacao guiada |
| automation_potential | Baixo (requer julgamento humano) |
| executor | Humano (comunicador) |
| precondition | Sequencia sendo publicada |
| output | Stories intermediarios adaptados; respostas no inbox |
| guardrails | O ouro esta no inbox — CADA mensagem e oportunidade |
| decision_rule | RD-08 |

**Acoes durante publicacao:**
- Ler respostas que chegam
- Usar prints de respostas como stories (Conversa sem Privacidade)
- Responder perguntas interessantes publicamente
- Adaptar proximo story com base no que a audiencia esta dizendo
- Identificar leads quentes para SPIN no inbox

> Fonte: [SOURCE: S10X-Core > Criando as sequencias]

## Parte 4: Debriefing e Amplificacao

### Step 4.1: Debriefing (Apos 24 horas) [GATE]

**Descricao:** Analisar resultados da sequencia e documentar aprendizados.

| Campo | Valor |
|-------|-------|
| cognitive_type | Analise |
| automation_potential | Alto (app pode calcular metricas) |
| executor | Humano com App S10X |
| precondition | Sequencia publicada ha 24+ horas |
| output | Relatorio de debriefing com metricas |
| guardrails | NAO analisar numeros sozinhos — analisar contexto (comentarios, sentimento) |
| decision_rule | RD-09 |

**Metricas a registrar:**
1. Views primeiro story
2. Views ultimo story
3. Taxa de retencao (ultimo/primeiro) — 34% = excelente; 16-20% = media
4. Total respostas inbox (somar todos os CTAs)
5. Para stories com links: views, cliques, CTR%, leads, vendas

**GATE:** Sem debriefing = voar cego. Debriefing alimenta o ciclo de Analise Pragmatica.

> Fonte: [SOURCE: S10X-Operacional > Debriefing]

### Step 4.2: Decidir Repetir ou Amplificar

**Descricao:** Com base no debriefing, decidir se a sequencia sera repetida, amplificada em outros formatos, ou descartada.

| Campo | Valor |
|-------|-------|
| cognitive_type | Decisao estrategica |
| automation_potential | Baixo |
| executor | Humano |
| precondition | Debriefing completo (Step 4.1) |
| output | Decisao: repetir / amplificar / descartar |
| guardrails | Sequencias podem ser repetidas 30-40x/ano; amplificar = converter em Reels, ads, podcasts |
| decision_rule | RD-10 |

**Decisao:**
- **Retencao alta + inbox alto** → REPETIR (agendar novamente) + AMPLIFICAR (converter em outro formato)
- **Retencao media + inbox medio** → REPETIR com ajustes nos dispositivos
- **Retencao baixa + inbox baixo** → DESCARTAR tema ou repensar sequencia completamente

> Fonte: [SOURCE: S10X-Core > A visao geral do metodo] + [SOURCE: S10X-Operacional > Debriefing]

## Regras de Decisao

| ID | Condicao | Acao |
|----|----------|------|
| RD-01 | Sem ideias de tema | Use as 4 fontes (Categorias, Identificacao, Urgencia Oculta, Tema Livre) — meta 70+ temas |
| RD-02 | Qual tipo de sequencia usar? | 90% engajamento/consciencia/caixinha; 10% venda/aquecimento |
| RD-03 | Qual cadencia? | Aquecimento/Pre-Abertura = SEMPRE alta; Caixinha = baixa; demais = depende do tema |
| RD-04 | Escrever copy | Copy PRIMEIRO, dispositivos DEPOIS; esqueleto (primeiro, ultimo, 5 chave) antes do meio |
| RD-05 | Quantos dispositivos? | Minimo 5 por sequencia; 3+ por story individual = excelente |
| RD-06 | CTA no primeiro story | OBRIGATORIO (enquete, caixinha, ou inbox); algoritmo penaliza stories sem interacao |
| RD-07 | Como postar link de venda | Link Oculto (D36): "Assista essa aula" > "Compre meu curso aqui" — 10x mais cliques |
| RD-08 | O que fazer com inbox durante publicacao | Monitorar, usar prints como stories, identificar leads para SPIN |
| RD-09 | Quando fazer debriefing | SEMPRE apos 24h; registrar: retencao, inbox, CTR |
| RD-10 | Repetir ou amplificar? | Retencao alta → repetir 30-40x/ano + converter em Reels/ads |
