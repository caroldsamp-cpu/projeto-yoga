## Como usar

1. Copie este checklist para cada nova sequencia de Stories
2. Siga as 4 partes em ordem
3. Items com GATE sao bloqueantes — nao avance sem completar
4. Parte 4 (Debriefing) se aplica APOS publicacao (24h depois)

## Parte 1: Planejamento da Sequencia

### Step 1.1: Tema [GATE]

- [ ] Tema definido em 1 frase: _______________________
- [ ] Fonte do tema (marque UMA):
  - [ ] Categorias (5 categorias do nicho × ideias)
  - [ ] Situacoes de Identificacao (dia-a-dia que gera identificacao)
  - [ ] Audiencia/Urgencia Oculta (problema que nao buscam pelo nome)
  - [ ] Tema Livre (ideia espontanea)
- [ ] Tema gera CONVERSA (nao e tutorial generico)
- [ ] Tema conecta com pelo menos 1 produto/servico (mesmo que indiretamente)

**GATE:** Sem tema = sem sequencia. O tema e o combustivel.

> Fonte: [SOURCE: S10X-Core > Ideias fortes e infinitas]

### Step 1.2: Tipo de Sequencia

- [ ] Tipo selecionado (marque UM):
  - [ ] Engajamento Puro
  - [ ] Consciencia de Produto
  - [ ] Caixinha por Tema
  - [ ] Caixinha Geral
  - [ ] Venda de Produto Digital
  - [ ] Venda de Produto Fisico
  - [ ] Aquecimento para Pico de Vendas
  - [ ] Pre-Abertura
  - [ ] Publicidade
  - [ ] Promocao de Parceiro
  - [ ] Evento
  - [ ] Divulgacao de Conteudo
- [ ] Confirmei: NAO estou fazendo sequencia de venda quando deveria ser engajamento (regra 90/10)

> Fonte: [SOURCE: S10X-Core > A logica das sequencias]

### Step 1.3: Cadencia e Data

- [ ] Cadencia definida:
  - [ ] Alta (postando muitas vezes ao longo do dia)
  - [ ] Baixa (poucos stories manha + noite)
- [ ] Confirmei: Se tipo = Aquecimento ou Pre-Abertura → cadencia DEVE ser alta
- [ ] Data agendada: ___/___/______
- [ ] Responsavel pela publicacao definido

> Fonte: [SOURCE: S10X-Core > A logica das sequencias]

## Parte 3: Publicacao e Monitoramento

### Step 3.1: Publicacao

- [ ] Status no App atualizado: Preenchido → Publicado
- [ ] Stories publicados na ordem planejada
- [ ] Cadencia mantida (alta = ao longo do dia; baixa = manha + noite)
- [ ] Nao alterei a ordem planejada

### Step 3.2: Monitoramento do Inbox

- [ ] Monitorei inbox durante a publicacao
- [ ] Usei prints de respostas como stories (Conversa sem Privacidade)
- [ ] Respondi perguntas interessantes publicamente
- [ ] Identifiquei leads quentes para SPIN no inbox
- [ ] Adaptei stories intermediarios com base no feedback

> Fonte: [SOURCE: S10X-Core > Criando as sequencias]

## Parte 4: Debriefing (Apos 24 horas) [GATE]

### Step 4.1: Metricas

- [ ] Esperei 24 horas apos publicacao
- [ ] Views primeiro story: _____
- [ ] Views ultimo story: _____
- [ ] Taxa de retencao: _____% (ultimo/primeiro)
  - [ ] 34%+ = Excelente
  - [ ] 16-20% = Media
  - [ ] <16% = Fraca
- [ ] Total respostas inbox: _____
- [ ] Para stories com links:
  - [ ] Views: _____ | Cliques: _____ | CTR: _____% | Leads: _____ | Vendas: _____

**GATE:** Sem debriefing = voar cego. Sem dados = sem melhoria.

> Fonte: [SOURCE: S10X-Operacional > Debriefing]

### Step 4.2: Decisao

- [ ] Decisao tomada (marque UMA):
  - [ ] **REPETIR + AMPLIFICAR** (retencao alta + inbox alto) → agendar nova data + converter em Reels/ads
  - [ ] **REPETIR COM AJUSTES** (retencao media) → ajustar dispositivos e reagendar
  - [ ] **DESCARTAR** (retencao baixa + inbox baixo) → repensar tema ou sequencia completamente
- [ ] Se amplificar: formato escolhido (Reels / Ad / Podcast / Live / Carrossel): _____
- [ ] Aprendizado registrado: ___________________________

> Fonte: [SOURCE: S10X-Core > A visao geral do metodo] + [SOURCE: S10X-Operacional > Debriefing]

## Regras de Decisao — Referencia Rapida

| ID | Condicao | Acao |
|----|----------|------|
| RD-01 | Sem ideias de tema | 4 fontes: Categorias, Identificacao, Urgencia Oculta, Tema Livre (meta 70+) |
| RD-02 | Qual tipo de sequencia? | 90% engajamento/consciencia/caixinha; 10% venda/aquecimento |
| RD-03 | Qual cadencia? | Aquecimento/Pre-Abertura = SEMPRE alta; Caixinha = baixa |
| RD-04 | Como escrever copy? | Copy PRIMEIRO, dispositivos DEPOIS; esqueleto antes do meio |
| RD-05 | Quantos dispositivos? | Min 5/sequencia; 3+/story = excelente |
| RD-06 | CTA no primeiro story? | OBRIGATORIO — algoritmo penaliza sem interacao |
| RD-07 | Como postar link? | Link Oculto: "Assista aula" > "Compre curso" (10x mais cliques) |
| RD-08 | O que fazer com inbox? | Monitorar, usar prints, identificar leads para SPIN |
| RD-09 | Quando debriefing? | SEMPRE apos 24h |
| RD-10 | Repetir ou descartar? | Retencao alta → repetir 30-40x/ano + amplificar |
