## Story

**As a** developer working on NamaSer ecosystem sites,
**I want** a complete, documented design system with extracted tokens from the approved LP visual,
**so that** all current and future NamaSer pages (LP, Primeiro Passo, Oraculo, Entrada Sadhana) share a consistent, professional, and accessible visual identity.

## Acceptance Criteria

1. All design tokens extracted from approved LP: colors, typography, spacing, borders, shadows, animations
2. Tokens exported as CSS Custom Properties (tokens.css)
3. Tokens exported in W3C DTCG format (tokens.json)
4. Color palette includes: primary, secondary, accent, neutral, semantic (success, warning, error, info) — all light/luminous
5. Typography system defines: font families, scale (sizes), weights, line-heights, letter-spacing
6. Spacing system defines: base unit and full scale (4px or 8px based)
7. All color combinations pass WCAG AA contrast ratio (minimum 4.5:1 for text)
8. Design System structure organized following Atomic Design principles
9. Documentation includes visual examples for each token category
10. DS armazenado em `design-system-v2/` na raiz do projeto
11. README with usage instructions for developers
12. DS armazenado em `design-system-v2/` na raiz do projeto

### Naming Convention para tokens
Prefixo: `--ns-` (NamaSer)
```
--ns-color-primary
--ns-color-secondary
--ns-color-accent
--ns-color-bg
--ns-color-text
--ns-font-family-serif
--ns-font-family-sans
--ns-font-size-xs ... --ns-font-size-4xl
--ns-spacing-xs ... --ns-spacing-4xl
--ns-radius-sm ... --ns-radius-xl
--ns-shadow-sm ... --ns-shadow-xl
```
