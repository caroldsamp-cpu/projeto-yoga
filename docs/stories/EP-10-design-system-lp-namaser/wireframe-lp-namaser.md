## Visão Geral da Estrutura

A LP segue um fluxo de conversão adaptado ao tom NamaSer: **sem urgência, sem culpa, baseado em convite**. A estrutura usa o padrão de copy da marca:

1. Dor real da rotina
2. Reframe sem culpa
3. Prática simples e aplicável
4. Convite claro para próximo passo

**Total de seções:** 12
**CTAs distribuídos:** 5 pontos estratégicos ao longo da página
**Tempo estimado de leitura:** 4-6 minutos (scroll completo)

## S1. HERO — Proposta de Valor

### Propósito
Comunicar instantaneamente o que é o NamaSer e para quem é. Capturar atenção emocional nos primeiros 3 segundos.

### Rationale de Conversão
> O hero é o filtro: quem se identifica continua rolando. A proposta de valor deve ser emocional (não funcional) e usar a linguagem real do público. Referência: gabydrummondyoga.com.br usa headline emocional + CTA de benefício.

### Wireframe

```
+----------------------------------------------------------+
|  [Nav: Logo NamaSer à esquerda]         [CTA pequeno]    |
+----------------------------------------------------------+
|                                                          |
|  [Overline: pequeno, serif, italic]                      |
|  Yoga na vida real. Sem performance.                     |
|                                                          |
|  [Headline principal: display serif, grande]             |
|  Construa seu Sadhana                                    |
|  com constância simples                                  |
|                                                          |
|  [Subtítulo: sans, tom mais leve]                        |
|  Um ritual diário de presença para mulheres que querem   |
|  leveza e calma — sem mudar toda a rotina.               |
|                                                          |
|  [CTA #1: botão accent, grande, arredondado]             |
|  "Quero conhecer o NamaSer"                              |
|                                                          |
|  [Micro-texto abaixo do CTA]                             |
|  Sem compromisso. Descubra no seu tempo.                 |
|                                                          |
|  [Imagem/visual: foto da Gaby em prática real,           |
|   ambiente claro, natureza da Serra, luz natural]        |
|                                                          |
+----------------------------------------------------------+
```

### Direção de Copy
- **Headline:** Usar "Construa seu Sadhana" (promessa central) + "constância simples" (diferencial)
- **Subtítulo:** Endereçar ICP diretamente — "mulheres", "rotina", "sem mudar tudo"
- **CTA:** Tom de convite, não de venda — "Quero conhecer" (não "Compre agora")
- **Overline:** Posicionar a categoria sem jargão

### Notas Mobile
- Headline em 2-3 linhas máximo
- CTA ocupa largura total (touch target 48px altura mínima)
- Imagem abaixo do texto no mobile (não ao lado)
- Nav colapsa em hamburger

## S2. DOR — Identificação

### Propósito
Criar espelho emocional. A visitante lê e pensa "isso sou eu". Validar a dor sem culpa.

### Rationale de Conversão
> Seção de identificação é o gatilho de engajamento. Se a pessoa se vê na descrição, o comprometimento emocional com a página aumenta. Posicionada logo após o hero para capturar quem continuou rolando. Baseado nas dores reais da pesquisa de 55 leads.

### Wireframe

```
+----------------------------------------------------------+
|                                                          |
|  [Overline: serif italic]                                |
|  Você se reconhece?                                      |
|                                                          |
|  [Grid de 3-4 cards, texto curto em cada]                |
|                                                          |
|  +------------------+  +------------------+              |
|  | "A mente não      |  | "Começo algo     |              |
|  |  para. Nem         |  |  e paro. Começo  |              |
|  |  pra dormir."      |  |  e paro."        |              |
|  +------------------+  +------------------+              |
|                                                          |
|  +------------------+  +------------------+              |
|  | "Queria um         |  | "Sei que preciso |              |
|  |  momento só        |  |  desacelerar,    |              |
|  |  meu no dia."      |  |  mas como?"      |              |
|  +------------------+  +------------------+              |
|                                                          |
|  [Texto de transição: serif, tom suave]                  |
|  Se você sentiu isso, você não está sozinha.             |
|  E não precisa mudar tudo para mudar algo.               |
|                                                          |
+----------------------------------------------------------+
```

### Direção de Copy
- Usar frases reais da pesquisa (33/55 "mais momentos só para mim", 32/55 "desacelerar a mente")
- Tom: observação, não diagnóstico — "Você se reconhece?" (não "Você sofre de...")
- Transição: validação + reframe sutil — "não precisa mudar tudo"

### Notas Mobile
- Cards empilham 1 coluna
- Texto dos cards com font-size legível (16px mínimo)
- Espaçamento generoso entre cards (24px gap)

## S3. REFRAME — Mudança de Lente

### Propósito
Introduzir a perspectiva NamaSer: constância simples > intensidade. Desconstruir a crença de que yoga exige muito.

### Rationale de Conversão
> Após identificar a dor, o reframe quebra a objeção principal ("não tenho tempo", "não vou conseguir manter"). Posicionada antes da apresentação do produto para preparar o terreno emocional. Segue o padrão: dor → reframe → solução.

### Wireframe

```
+----------------------------------------------------------+
|                                                          |
|  [Bloco com fundo levemente diferenciado — creme/sage]   |
|                                                          |
|  [Citação grande: display serif]                         |
|  "Constância simples transforma mais                     |
|   que intensidade ocasional."                            |
|                                                          |
|  [Texto de apoio: sans, 2-3 linhas]                      |
|  Você não precisa de uma hora no tapete.                 |
|  Não precisa ser flexível. Não precisa parar tudo.       |
|  Precisa de um caminho possível — e alguém               |
|  que caminhe com você.                                   |
|                                                          |
+----------------------------------------------------------+
```

### Direção de Copy
- Claim aprovado #1 como citação de destaque
- Desconstruir 3 mitos em sequência curta ("não precisa de X, não precisa de Y, não precisa de Z")
- Fechar com a ideia de acompanhamento (comunidade + Gaby)
- Tom: Gaby falando diretamente — conversacional, como em aula

### Notas Mobile
- Citação ocupa tela cheia com padding generoso
- Background sutil marca separação visual
- Font-size da citação: clamp(1.5rem, 4vw, 2.5rem)

## S5. COMO FUNCIONA — O Método

### Propósito
Mostrar clareza e simplicidade do método. Resolver a objeção "não sei como funciona" (18/55 leads).

### Rationale de Conversão
> Após a identificação emocional, a mente racional precisa de clareza. "Como funciona?" é a pergunta natural. 3 passos simples eliminam a barreira cognitiva. Referência: sites de conversão eficientes usam "3 steps" para transmitir simplicidade. Adaptado sem agressividade.

### Wireframe

```
+----------------------------------------------------------+
|                                                          |
|  [Título: serif]                                         |
|  Como funciona o NamaSer                                 |
|                                                          |
|  [Subtítulo: sans, leve]                                 |
|  Seu caminho em 3 passos simples                         |
|                                                          |
|  [3 cards horizontais (desktop) / verticais (mobile)]    |
|                                                          |
|  +----------------+  +----------------+  +---------------+|
|  |   [Ícone 1]    |  |   [Ícone 2]    |  |   [Ícone 3]  ||
|  |                 |  |                 |  |              ||
|  |   Passo 1       |  |   Passo 2       |  |   Passo 3   ||
|  |   Comece pelo   |  |   Siga o        |  |   Evolua     ||
|  |   Desafio       |  |   Caminho       |  |   no seu     ||
|  |   Sadhana       |  |   de 30 dias    |  |   ritmo      ||
|  |                 |  |                 |  |              ||
|  |  Escolha 1      |  |  Um guia leve   |  |  5 trilhas   ||
|  |  ritual de 3-5  |  |  com no máximo  |  |  completas   ||
|  |  min e pratique |  |  2 aulas por    |  |  pra quando  ||
|  |  por 7 dias.    |  |  semana, no     |  |  você quiser ||
|  |  Sem tapete,    |  |  seu tempo.     |  |  ir mais     ||
|  |  sem roupa      |  |                 |  |  fundo.      ||
|  |  especial.      |  |                 |  |              ||
|  +----------------+  +----------------+  +---------------+|
|                                                          |
|  [Texto de apoio: sans, sutil]                           |
|  47% das alunas sentem leveza na primeira prática.       |
|                                                          |
+----------------------------------------------------------+
```

### Direção de Copy
- 3 passos que refletem a jornada real: Desafio 7 dias → Caminho 30 dias → Trilhas completas
- Passo 1 mata a objeção "não tenho tempo" (ritual de 3-5 min, sem tapete)
- Passo 2 mostra que existe estrutura sem pressão ("no máximo 2 aulas/semana")
- Passo 3 abre para profundidade sem forçar ("quando quiser ir mais fundo")
- Dado real (47%) como prova sutil no final
- Tom: explicativo, sem complicação — como Gaby explicaria para uma amiga

### Notas Mobile
- Cards empilham verticalmente
- Ícones centralizados
- Numeração visual clara (1, 2, 3)
- Gap entre cards: 24px

## S6. O QUE VOCÊ RECEBE — Ritual + Trilhas + Bônus

### Propósito
Mostrar o conteúdo concreto do produto organizado pela jornada: ritual diário (entrada) → trilhas de yoga (evolução) → bônus (apoio contínuo). Justificar o valor com quantidade e qualidade.

### Rationale de Conversão
> Stack de valor — mostrar o que contém o produto cria percepção de "muito por pouco". Posicionada após "como funciona" porque a pessoa já entende o método e agora quer saber o que recebe. A organização em 3 camadas (ritual → trilhas → bônus) reforça que a entrada é simples (ritual de minutos) e a profundidade vem naturalmente. Adaptado do padrão de conversão sem ser agressivo (sem preços individuais falsos).

- **Hierarquia narrativa:** Ritual (entrada fácil) → Trilhas (profundidade) → Bônus (apoio)

- **Bloco 2 (Trilhas):** Accordion — "quando quiser ir mais fundo" (sem pressão pra fazer tudo)
- **Bloco 3 (Bônus):** Cards — valor extra percebido (Spotify, Oráculo, Entrada Sadhana)
- Trilhas 2-3 seguem ciclos lunares (diferencial único do NamaSer)
- Mensagem central: "Comece pelo ritual. As aulas vêm quando você estiver pronta."
- Resumo visual no final como "snapshot" do valor total
- Tom: descritivo, concreto, sem exagero

### Notas Mobile
- Accordion é ideal para mobile (economiza espaço)
- Resumo visual em grid 2x2
- Touch targets confortáveis nas abas do accordion (48px altura)

## Notas de Responsividade (Mobile-First)

### Breakpoints
- **Mobile:** < 640px (layout padrão — tudo empilhado)
- **Tablet:** 640px - 1024px (grids de 2 colunas onde aplicável)
- **Desktop:** > 1024px (layouts lado a lado, grids maiores)

### Regras globais mobile
1. **Tipografia:** clamp() para escala fluida — nunca quebrar em telas pequenas
2. **CTAs:** full-width no mobile, tamanho de toque 48px mínimo
3. **Imagens:** aspect-ratio preservado, lazy loading abaixo do fold
4. **Espaçamento:** padding mínimo 16px lateral, 32px entre seções
5. **Navegação:** hamburger menu no mobile, nav fixa no desktop
6. **Accordion/FAQ:** essencial para mobile — economiza scroll
7. **Carousel:** swipe horizontal para depoimentos
8. **Velocidade:** otimizar imagens, preload de fontes críticas

### Hierarquia visual mobile
- Hero: headline grande → subtítulo → CTA (nada mais)
- Seções: título → conteúdo → CTA (quando houver)
- Eliminar elementos decorativos que não agregam valor no mobile
