## User Story

**Como** desenvolvedor do squad light-copy-kb,
**Quero** que os 3.581 chunks normalizados tenham os campos `techniques`, `literary_elements` e `persuasion_drivers` preenchidos via classificação LLM,
**Para que** o retrieval do agente de rewrite seja preciso e rastreável, citando técnicas de Light Copy com base no conteúdo real de cada chunk.

---

## Contexto

O pipeline de normalização de transcrições foi concluído e produziu 3.581 chunks estruturados em `squads/light-copy-kb/data/normalized-chunks.jsonl`. Contudo, uma validação de qualidade revelou que **82% dos chunks (2.924/3.581) estão com o campo `techniques` vazio**, e 100% estão sem `literary_elements` e `persuasion_drivers`.

O script `normalize-transcripts.js` usa heurísticas simples (matching de keywords) que não capturam a semântica real do texto. A solução é um script de enriquecimento LLM-based que leia cada chunk, classifique seu conteúdo conforme a taxonomia controlada (`metadata-taxonomy.yaml`) e reescreva o JSONL enriquecido. Após o enriquecimento, o Chroma deve ser re-indexado com os novos metadados.

**Gap identificado (validação `kb-quality-report.json`):**
- `techniques`: 82% vazio (2.924/3.581 chunks)
- `literary_elements`: ~100% vazio
- `persuasion_drivers`: ~100% vazio
- `confidence_score` médio: 0.52 (baixo — classificação sem LLM)

## Scope

### Inclui
- Script `squads/light-copy-kb/scripts/enrich-techniques.js` que processa o JSONL chunk por chunk via Anthropic Claude API
- Prompt few-shot baseado na taxonomia controlada (`metadata-taxonomy.yaml`)
- Atualização do `data/normalized-chunks.jsonl` com campos enriquecidos
- Re-geração do `data/kb-quality-report.json` com métricas pós-enriquecimento
- Re-ingestão no Chroma (`data/chroma/`) com metadados atualizados
- Mecanismo de checkpoint (resume) para não reprocessar chunks já enriquecidos

- Criação do agente de rewrite (S-09.2 — próxima story)
- Alteração do schema `schema-light-copy.json` (os campos já existem, só estão vazios)
- Nova ingestão de transcrições
- Fine-tuning de modelo
- Interface de usuário
