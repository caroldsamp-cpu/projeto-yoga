# S-03.5 - Criar Checklist "Isto e Light Copy?"

**Epic:** EP-03 - Framework de Light Copy
**Story ID:** S-03.5
**Tipo:** Documentacao / Validacao
**Prioridade:** MUST
**Estimativa:** 3 pontos
**Status:** [ ] Nao Iniciada
**Dependencias:** S-03.1 (Light Copy Pattern), S-03.2 (Vocabulario)

---

## User Story

**Como** criadora de conteudo do projeto NamaSer,
**Quero** ter um checklist com 10+ criterios para validar se uma peca de conteudo e realmente Light Copy,
**Para que** nenhum conteudo publicado traia a filosofia da marca, mesmo quando criado sob pressao ou por terceiros.

---

## Objetivo

Criar um instrumento pratico de validacao - um checklist rapido e objetivo que qualquer pessoa possa aplicar a qualquer peca de conteudo (post, video, email, mensagem) para verificar se ela atende aos principios do Light Copy. O checklist funciona como um "filtro de qualidade" que protege a integridade da comunicacao do NamaSer.

---

## Contexto

A maior ameaca ao Light Copy nao e o desconhecimento - e a pressa. Quando ha pressao para publicar, a tendencia natural e recorrer a formulas de Heavy Copy (urgencia, escassez, promessas grandes). Este checklist e o "guardioes do tom" - rapido o suficiente para usar em 2 minutos, robusto o suficiente para pegar desvios sutis.

**Analogia:** E como um checklist de voo - o piloto sabe voar, mas o checklist impede que a rotina gere erros.

---

## Scope

### Inclui
- Checklist principal com 10+ criterios de validacao Sim/Nao
- Sistema de pontuacao (quantos "sim" sao necessarios para aprovar)
- Secao "Red Flags" - sinais de alerta que indicam Heavy Copy
- Secao "Green Flags" - sinais de que esta no caminho certo
- Guia rapido de correcao (se falhou em X, faca Y)
- Exemplos de conteudo que PASSA e que NAO PASSA no checklist
- Versao resumida (pocket) para uso diario rapido

### Nao Inclui
- Reescrita completa de conteudos (referencia para S-03.1)
- Analise de creators externos (isso e S-03.6/S-03.7)
- Metricas de performance de conteudo (isso e EP-05)

### Fase 2: Construcao do Checklist Principal
- [ ] 2.1 Formatar checklist com formato Sim/Nao por criterio
- [ ] 2.2 Criterios de TOM E LINGUAGEM (minimo 3):
  - [ ] "A mensagem soa como uma conversa, nao como uma venda?"
  - [ ] "O vocabulario esta alinhado com a paleta de palavras da Gaby?"
  - [ ] "O ritmo tem pausas intencionais (nao e texto corrido apressado)?"
  - [ ] "A linguagem e poetica sem ser rebuscada?"

- [ ] 2.3 Criterios de ESTRUTURA (minimo 3):
  - [ ] "A mensagem segue ao menos 4 dos 6 passos do Light Copy Pattern?"
  - [ ] "Ha uma progressao natural (abertura â†’ desenvolvimento â†’ encerramento)?"
  - [ ] "O CTA e um convite, nao uma ordem?"
  - [ ] "A mensagem fecha com reforco emocional, nao com urgencia?"

- [ ] 2.4 Criterios de INTENCAO (minimo 2):
  - [ ] "A mensagem gera identificacao antes de propor acao?"
  - [ ] "O leitor se sente validado, nao culpado?"
  - [ ] "A mensagem daria vontade de compartilhar com uma amiga?"

- [ ] 2.5 Criterios de ETICA (minimo 2):
  - [ ] "A mensagem esta livre de urgencia artificial e escassez falsa?"
  - [ ] "A mensagem respeita a autonomia do leitor (nao manipula)?"
  - [ ] "A promessa e realista e verificavel?"

### Fase 3: Sistema de Pontuacao
- [ ] 3.1 Definir regra de aprovacao (ex: 8/10 criterios = aprovado)
- [ ] 3.2 Definir criterios OBRIGATORIOS (que sozinhos reprovam se falharem)
- [ ] 3.3 Definir niveis: Light Copy Puro (10/10), Light Copy Aceitavel (8/10), Precisa Revisao (6-7/10), Heavy Copy (<6/10)
- [ ] 3.4 Criar escala visual simples para uso rapido

### Fase 4: Red Flags e Green Flags
- [ ] 4.1 Criar lista de 10+ RED FLAGS (sinais de Heavy Copy)
  - [ ] Palavras de urgencia: "corra", "ultimas vagas", "so hoje"
  - [ ] Promessas exageradas: "transforme sua vida", "resultados garantidos"
  - [ ] Pressao emocional: "voce vai se arrepender", "nao perca"
  - [ ] Comparacao: "seja como fulana", "nao fique pra tras"
  - [ ] Gatilho de culpa: "ate quando voce vai..."
  - [ ] Tom imperativo: "voce PRECISA", "faca AGORA"
  - [ ] Superlatives vazios: "o melhor", "unico", "revolucionario"

- [ ] 4.2 Criar lista de 10+ GREEN FLAGS (sinais de Light Copy)
  - [ ] Convite suave: "perceba", "experimente", "quando sentir vontade"
  - [ ] Validacao: "tudo bem", "no seu ritmo", "sem pressa"
  - [ ] Premissa logica: "porque...", "quando a gente entende que..."
  - [ ] Sensorialidade: descricoes de sensacoes fisicas
  - [ ] Poeticidade: metaforas naturais, linguagem bonita
  - [ ] Espaco: pausas, respiros, silencios no texto
  - [ ] Circularidade: encerramento que retoma abertura

### Fase 5: Guia de Correcao Rapida
- [ ] 5.1 Para cada criterio do checklist, escrever:
  - [ ] "Se falhou neste criterio, experimente..."
  - [ ] Tecnica rapida de correcao (1-2 frases)
  - [ ] Exemplo de antes/depois

- [ ] 5.2 Criar secao "Transformando Heavy em Light" com 3 exemplos completos
  - [ ] Heavy Copy original
  - [ ] O que esta errado (criterios falhados)
  - [ ] Light Copy corrigido
  - [ ] Por que a versao corrigida funciona

### Fase 6: Versao Pocket e Formatacao
- [ ] 6.1 Criar versao resumida (checklist em 1 pagina, sem explicacoes)
- [ ] 6.2 Formatar documento completo com indice
- [ ] 6.3 Incluir instrucoes de uso no inicio
- [ ] 6.4 Revisar completude e coerencia

## Regras Sadhana

### SEMPRE
- O checklist deve ser tao leve quanto o conteudo que ele valida
- Cada criterio deve ter justificativa clara (por que isso importa)
- Incluir o "como corrigir", nao apenas o "esta errado"
- Manter tom de guia, nao de policia

### NUNCA
- Criar tantas regras que paralise a criacao
- Esquecer que Light Copy tem excepcoes e nuances
- Usar o checklist como arma de critica, mas sim como ferramenta de melhoria
- Ignorar que conteudo autentico as vezes quebra regras propositalmente
